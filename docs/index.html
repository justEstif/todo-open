todo.open

A local-first task server with an open API.
Your tasks live on disk. Any tool — CLI, agent, or script — talks to the same endpoint. 

# human creates a task
$ todoopen task create --title "Refactor auth module" --priority high
Created task_a1b2: Refactor auth module

# agent picks it up and claims it
$ curl -s localhost:8080/v1/tasks/next | jq '.id'
"task_a1b2"
$ curl -s -X POST localhost:8080/v1/tasks/task_a1b2/claim -H 'X-Idempotency-Key: run-42'

# human checks progress
$ todoopen task list
ID          TITLE                  STATUS       HELD BY
task_a1b2   Refactor auth module   in_progress  agent-claude (expires 4m)

How it works

One server. One API. Everything else is a client.

  ┌──────────────────────────────────────────┐
  │           .todoopen/tasks.jsonl          │  ← plain text, yours forever
  └──────────────────┬───────────────────────┘
                     │
          ┌──────────▼──────────┐
          │   todoopen server   │  ← local HTTP, :8080
          │   (REST + SSE)      │
          └──┬──────┬───────────┘
             │      │
    ┌─────────┘      └──────────────┐
    │                               │
┌───▼───┐   ┌──────────┐   ┌───────▼──────┐
│  CLI  │   │  web UI  │   │  AI agents   │
│       │   │          │   │  /tasks/next │
└───────┘   └──────────┘   └──────────────┘
Plain files

Tasks are stored as tasks.jsonl — one JSON object per line. No database, no daemon to manage, no lock-in. Open the file in any editor.

Open API

Full REST + Server-Sent Events API. Any tool that can speak HTTP can create, claim, watch, and complete tasks. Inspect it with curl.

Agent-safe

Built-in lease system prevents two agents from claiming the same task. Idempotency keys make retries safe. Human can always override.

Install

Pick your preferred package manager.

$ npm install -g @justestif/todo-open
$ mise use -g go:github.com/justEstif/todo-open/cmd/todoopen@latest
$ mise reshim
$ git clone https://github.com/justEstif/todo-open.git
$ cd todo-open
$ go build ./cmd/todoopen ./cmd/todoopen-server

Then start the server and open the web UI:

$ todoopen web
Server listening on http://127.0.0.1:8080
Workspace: /home/user/projects/myapp/.todoopen

Adapters

Adapters are separate binaries that extend sync and view behavior. Install only what you need.

Adapter Kind What it does
todoopen-adapter-sync-git sync Push/pull tasks.jsonl to a git repo
todoopen-adapter-sync-s3 sync Sync workspace to an S3 bucket
todoopen-adapter-view-markdown view Render tasks as a TASKS.md file
build your own sync view Any language, any backend — just speak the protocol

Install an adapter the same way as the CLI:

$ npm install -g @justestif/todoopen-adapter-sync-git
$ mise use -g go:github.com/justEstif/todo-open/adapters/todoopen-adapter-sync-git@latest
$ mise reshim
Adapters are completely decoupled — they communicate over stdin/stdout JSON and can be written in any language. See adapters.md ↗ to build your own.

Agents

todo.open is built for human-agent collaboration. Agents get primitives; humans stay in control.

The agent workflow: fetch the next unclaimed task, claim it with a lease, send heartbeats while working, complete or release when done.

# 1. fetch the next open task
$ curl -s localhost:8080/v1/tasks/next
{ "id": "task_a1b2", "title": "Refactor auth module", "status": "open" }

# 2. claim it (lease = 5 min, idempotent)
$ curl -s -X POST localhost:8080/v1/tasks/task_a1b2/claim \
    -H 'X-Idempotency-Key: run-42' \
    -d '{"assignee":"agent-claude"}'

# 3. heartbeat every 60s to keep the lease alive
$ curl -s -X POST localhost:8080/v1/tasks/task_a1b2/heartbeat

# 4. complete
$ curl -s -X POST localhost:8080/v1/tasks/task_a1b2/complete

Watch task activity in real time:

$ curl -s localhost:8080/v1/tasks/events
event: task.status_changed
data: {"id":"task_a1b2","status":"in_progress","assignee":"agent-claude"}

event: task.status_changed
data: {"id":"task_a1b2","status":"done"}
Lease system

Claims expire automatically. If an agent crashes, the task becomes available again. No manual cleanup.

Idempotent ops

Pass X-Idempotency-Key on mutating requests. Safe to retry without duplicating work.

Human override

Humans can always release an agent's claim. The API and CLI never block human intervention.

Discover the full agent contract at runtime: 

$ curl -s localhost:8080/v1/capabilities | jq .agent
{
  "endpoints": {
    "next":      "GET  /v1/tasks/next",
    "claim":     "POST /v1/tasks/{id}/claim",
    "heartbeat": "POST /v1/tasks/{id}/heartbeat",
    "complete":  "POST /v1/tasks/{id}/complete"
  },
  "lease_ttl_default_seconds": 300
}

Configure

Create .todoopen/config.toml in your workspace. Config is read fresh on every server start — no restart needed for adapter changes.

# .todoopen/config.toml — no adapters, just the server
[server]
  addr = "127.0.0.1:8080"
[adapters.git]
  bin = "todoopen-adapter-sync-git"

[adapters.git.config]
  remote = "origin"
  branch = "main"
[server]
  addr = "127.0.0.1:8080"

[adapters.git]
  bin = "todoopen-adapter-sync-git"

[adapters.git.config]
  remote = "${GIT_REMOTE}"   # expanded from env at runtime
  branch = "tasks"

[adapters.s3]
  bin = "todoopen-adapter-sync-s3"

[adapters.s3.config]
  bucket = "${S3_BUCKET}"
  region = "us-east-1"
Use ${VAR} in adapter config values — expanded from the environment at runtime so secrets never live in the file.
